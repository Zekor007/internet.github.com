Основи мережі Інтернет та протоколів

1. Основні терміни

IP-адреса

Унікальний ідентифікатор пристрою в мережі, що дозволяє пристроям знаходити і "спілкуватися" між собою в Інтернеті.

MAC-адреса

Фізична адреса мережевої карти, унікальна для кожного пристрою. Використовується на канальному рівні для ідентифікації.

DNS (Domain Name System)

Система, що перетворює доменні імена (наприклад, `google.com`) у відповідні IP-адреси.

LAN / WAN

  • LAN (Local Area Network) – локальна мережа в межах будівлі, офісу.
  • WAN (Wide Area Network) – глобальна мережа, наприклад, Інтернет.

2. Мережева адресація

DHCP (Dynamic Host Configuration Protocol)

Протокол, який автоматично призначає IP-адреси пристроям у мережі.

  • Статична резервація — IP-адреса закріплюється за конкретним пристроєм.

CIDR (Classless Inter-Domain Routing)

Метод представлення IP-адрес з маскою, наприклад: `192.168.0.1/24`.

3. NAT (Network Address Translation)

Технологія, яка переписує IP-адреси з локальних (сірі адреси) на глобальні (білі).

Види NAT:

  • Статичний — один локальний = один глобальний
  • Динамічний — IP береться з пулу доступних адрес
  • PAT (Port Address Translation) — багато локальних адрес → одна глобальна + порти

4. Протоколи передачі даних

TCP (Transmission Control Protocol)

  • Протокол на транспортному рівні
  • Надійний, гарантована доставка
  • Повільніший, перевіряє цілісність пакетів
  • Використовується для передачі важливих даних (вебсайти, email)

UDP (User Datagram Protocol)

  • Швидший, без перевірок доставки
  • Може втрачати дані, не гарантує послідовність
  • Підходить для відео, голосового зв'язку

5. Протоколи доступу до веб

HTTP (Hypertext Transfer Protocol)

  • Протокол для передачі вебсторінок (порт 80)
  • Не шифрує дані, може бути перехоплений

HTTPS (Secure HTTP)

  • Такий самий, але з шифруванням
  • Використовує SSL/TLS
  • Безпечніший, захищає особисті дані
